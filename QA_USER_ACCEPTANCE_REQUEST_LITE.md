# Process Coaching AI 사용자 관점 QA 요청서 (경량)

문서 버전: v1.0  
작성일: 2026-02-20

## 1) 목적
- 자동화 통과 확인이 아니라 실제 사용자 관점의 사용성/완결성 검증
- 중점: 처음 사용자 막힘, 오류 후 복구, 코칭 제안 실효성

## 2) 테스트 범위 (이번 릴리즈)
- 진입: Setup(L4/L5/L6) -> 메인 화면 진입
- 편집: 노드 추가/연결/라벨 수정/삭제, Undo/Redo
- 구조화: 역할 구분선(스윔레인) ON/OFF, 레인 추가
- 품질: L7 전체 검증, QualityDashboard, "해당 노드 보기"
- 코칭: 전체 흐름 검토, 채팅 Q&A, 맥락 제안
- 저장/완료: 중간저장(JSON), 복구, 완료하기 품질 게이트

## 3) 필수 시나리오 (8개)
1. 첫 진입 후 5분 내 초안 작성(노드 3개 이상 + 연결 + 중간저장)
2. Decision 분기 라벨을 이해하고 올바르게 입력
3. L7 검증 후 우선 수정 노드를 즉시 식별
4. 모호/복수 동작 라벨을 교정해 통과 상태로 개선
5. 스윔레인으로 역할 기준 재정리 성공
6. 코칭 제안을 실제 플로우에 반영 가능
7. 저장/복구/가져오기에서 데이터 유실 없음
8. 완료하기 게이트 이슈를 스스로 해소 후 완료

## 4) 환경
- Desktop: Chrome 최신, Edge 최신
- Mobile View: iPhone 14 / Galaxy S22 폭
- API 모드: live, mock 각각 최소 1회

## 5) 합격 기준
- P0: 0건
- P1: 0건(또는 승인권자 수용 명시)
- 필수 시나리오 1~8 전부 통과
- 사용자 관점 개선 제안 최소 5건

## 6) 결함 심각도
- P0: 데이터 유실/저장·복구 실패/완료 불가/핵심 진입 불가
- P1: 핵심 작업 성공률 저하(우회 어려움)
- P2: 우회 가능하나 혼란/품질 저하 큼
- P3: 경미한 UI/문구/일관성 이슈

- 심각도: P0/P1/P2/P3
- 환경: OS, 브라우저/버전, 해상도, 계정상태, API 모드

- 실제 결과:
- 재현율: 항상/간헐(%)
- 영향 범위:
- 임시 우회책:
- 증빙: 스크린샷/영상/콘솔/네트워크 로그

## 동료에게 전달할 메시지 (복붙용)
이번 QA는 자동화 재검증이 아니라 실사용자 관점 검증입니다.  
`QA_USER_ACCEPTANCE_REQUEST.md`(상세)와 `QA_USER_ACCEPTANCE_REQUEST_LITE.md`(경량) 기준으로 시나리오 1~8을 우선 수행해 주세요.  
특히 "처음 사용자 막힘", "오류 후 복구", "코칭 제안 실효성" 중심으로 확인 부탁드립니다.
